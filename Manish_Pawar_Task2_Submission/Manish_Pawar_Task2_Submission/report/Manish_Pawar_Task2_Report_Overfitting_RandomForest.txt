Overfitting means a machine learning model learns the training data too deeply, including small noise and unnecessary details, instead of learning the general pattern. In simple words, the model becomes very good at answering questions from the training data, but it performs poorly on new unseen data. This is similar to a student who memorises only the exact answers from a notebook but cannot solve a new question in the exam. In Decision Tree models, overfitting usually happens when the tree grows too deep. A deep tree keeps splitting the data again and again until it may remember individual training examples. Because of this, training accuracy can become very high, even 100%, but test accuracy may reduce.

Random Forest helps to solve overfitting by using many Decision Trees instead of only one tree. Each tree is trained on a different random sample of the data and also uses random features while making splits. After all trees make their predictions, the Random Forest takes the majority vote for classification. This voting process reduces the effect of one weak or overfitted tree. If one tree makes a wrong prediction, many other trees can correct it.

In this task, the Iris dataset was used to classify flowers into Setosa, Versicolor, and Virginica. A Random Forest model was trained and evaluated using accuracy, cross-validation, classification report, confusion matrix, feature importance, and model comparison. The final tuned Random Forest achieved excellent performance on the test data. Feature importance also showed that petal length and petal width are the most useful features for classifying Iris flowers. Overall, Random Forest is more stable and reliable than a single Decision Tree because it combines multiple trees and gives a stronger final prediction.
